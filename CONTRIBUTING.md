# 🤝 How to Contribute to This Project

Thank you for wanting to improve this repository! Here's how to do it.

---

## What kind of contributions are we looking for?

- 🧠 **New prompts** — Specific use cases (churn analysis, competitive intel, etc.)
- 🌍 **Translations** — Master prompt and guides in other languages
- 📊 **RICE framework variations** — Adaptations for different product types
- 🐛 **Fixes** — Errors in documentation or prompts that don't work well
- 💡 **Real examples** — Anonymized use cases that demonstrate the system's value

---

## Contribution process

1. **Fork** this repository
2. Create a descriptive branch: `feature/churn-analysis-prompt` or `fix/rice-formula-typo`
3. Make your changes with clear commits
4. Open a **Pull Request** describing:
   - What you changed and why
   - How you tested it (e.g., "Ran this prompt in NotebookLM with 10 feedback PDFs")
5. Wait for review — we respond within 48 hours

---

## Quality Standards

- Prompts must be **tested in NotebookLM** before submitting
- Documentation must be in **English or Spanish** (both are welcome)
- Follow the **existing file structure**
- Include a sample output whenever possible

---

## Code of Conduct

This is a professional collaboration space. Be respectful, constructive, and focused on value for the PM community.

---

## Questions?

Open an [Issue](../../issues) with the `question` label and we'll be happy to answer.
